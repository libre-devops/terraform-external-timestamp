# Terraform External Timestamp

Generate a timestamp at plan time (not apply time), cross platform, with a format you control.

## Overview

Terraform's built in `timestamp()` returns a fixed RFC3339 UTC string that is only known at apply.
This module runs an external program during **plan**, so the value is available to expressions that
must resolve before apply (a resource name suffix, a tag, a deployment id), and the format is yours.

It detects the host OS (see the [detect-os](https://github.com/libre-devops/terraform-external-detect-os)
module for the same mechanism) and then formats the time with `date` on Linux/macOS or `Get-Date` on
Windows. `check` blocks assert the OS is recognised and the timestamp is non-empty. No provider auth,
no resources created.

## Usage

```hcl
module "timestamp" {
  source  = "libre-devops/timestamp/external"
  version = "~> 4.0"

  linux_timestamp_format   = "+%Y-%m-%d"   # date(1) format, must begin with +
  windows_timestamp_format = "yyyy-MM-dd"  # Get-Date -Format
}

# module.timestamp.timestamp -> e.g. "2026-07-01"
```

## Developing

Local work needs **PowerShell 7+** and **[`just`](https://github.com/casey/just)**, because the recipes
wrap the [LibreDevOpsHelpers](https://www.powershellgallery.com/packages/LibreDevOpsHelpers)
PowerShell module. Install just with `brew install just`, or `uv tool add rust-just` then
`uv run just <recipe>`.

Run `just` to list recipes: `just update-ldo-pwsh`, `just validate` (fmt check, validate, tflint,
trivy), `just test` (native tests), `just pwsh-analyze`, `just e2e` (apply an example then destroy
it; local state, no cloud), and `just docs`. This module creates no cloud resources, so there is no
backend, no Azure login, and nothing to tear down. Releasing is also `just`:
`just increment-release [patch|minor|major]` bumps, tags, and publishes a GitHub release, and the
Terraform Registry picks up the tag.

## Security scan exceptions

This module is scanned with [Trivy](https://github.com/aquasecurity/trivy); HIGH and CRITICAL
findings fail the build. There are currently **no exceptions**: the module creates no cloud
resources, so there is no posture to waive. Waivers, if ever needed, live in
[`.trivyignore.yaml`](./.trivyignore.yaml) and are mirrored in a table here.
